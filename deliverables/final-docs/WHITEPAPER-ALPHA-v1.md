## 2. Problem Statement

The business already generates real economic value through:

- fiat inflow
- `Purchase Credit (PC)`
- `Sales Point (SP)`
- reward obligations
- payouts
- and digital product fulfillment

The core issue is not the absence of economic value.

The issue is that value still moves through fragmented operating loops rather than one controlled internal settlement design. That fragmentation reduces:

- value circulation inside the ecosystem
- policy visibility
- fairness control
- and control over how value exits the system

## 3. Phase 1 Positioning

Whitepaper v1 should define `ALPHA` in Phase 1 as:

- an internal settlement and rights layer
- policy-governed, not market-priced
- tied to existing business activity
- non-public and non-transferable by default
- designed to structure value before any later public-token path is considered

Phase 1 still does **not** imply:

- public token trading
- open transferability
- live market behavior
- automatic equivalence to later public assets such as `iBC` or `iBTC`
